Challenges

Initially, Permiso built custom solutions using AWS Lambda, DynamoDB, and PostgreSQL to process and store security event data. While this approach worked for their first few use cases, scaling became a major hurdle as new use cases and enterprise customers were added.

The custom-built solutions introduced several inefficiencies:

•

High Turnaround Time:

Every new use case required additional development effort, slowing down the deployment of new features.

•

Infrastructure Complexity:

Each use case needed a separate infrastructure design, making management cumbersome.

•

Data store Proliferation:

Data was carefully fragmented between multiple data stores to solve specific problems. Sometimes a single use case required the use of multiple stores at various stages of processing creating subtle data synchronization inconsistencies that required developer attention.

•

Increasing Costs:

As the number of use cases grew, the cost of maintaining and scaling infrastructure increased significantly.

•

Manual Scaling Burden:

Human intervention was required to manage and scale clusters, leading to operational inefficiencies.

How Mach5's All-in-one Search and Analytics Platform Transformed Permiso's Security Analytics

"By adopting Mach5, Permiso streamlined its data infrastructure, allowing all raw and processed event data to be stored in a single, centralized event store. This enabled multiple security analytics use cases without the need to constantly rethink infrastructure."
